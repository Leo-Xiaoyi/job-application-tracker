# Development Roadmap

## Phase 1: CLI Prototype

- Define job application data model
- Add application records
- List application records
- Update application status
- Delete application records
- Store data in local JSON file

## Phase 2: Backend API

- Create Express server
- Add REST API routes
- Implement CRUD endpoints
- Add validation and error handling

## Phase 3: Database Integration

- Design PostgreSQL schema
- Add Prisma ORM
- Replace JSON storage with database queries

## Phase 4: Frontend Dashboard

- Build application list page
- Build add application form
- Build status badges
- Build filters and dashboard statistics

## Phase 5: Authentication

- Add user model
- Add register and login
- Protect application routes
- Ensure users only access their own data

## Phase 6: AI Assistant

- Add job description analysis
- Extract required skills
- Match job requirements with user profile
- Generate cover letter drafts

## Phase 7: Testing, Deployment, and Portfolio Polish

- Add meaningful tests
- Deploy frontend and backend
- Add screenshots
- Polish README
- Prepare project explanation for interviews

## Personal emphasis (Week 0)

I am prioritising **Phase 6: AI Assistant** alongside solid full-stack foundations: JD analysis and cover letter drafts are the clearest signal for LLM application roles, but they only make sense once auth and persistence are in place (Phases 3–5).
